### Quick Fixes

- **color-contrast** (serious, 5 instances): Increase contrast ratio to ≥4.5:1 for normal text or ≥3:1 for large text. See Color Contrast Details below.
- **landmark-one-main** (moderate, 1 instances): Wrap the primary content area in a \`<main>\` element. This also resolves most \`region\` violations.
- **region** (moderate, 11 instances): Ensure all page content is inside a landmark region (\`<main>\`, \`<nav>\`, \`<header>\`, \`<footer>\`, or \`role="..."\`).

## Manual Testing Recommendations

Automated findings were limited to color contrast and landmark structure, so
manual effort should concentrate there first, with a baseline pass on
keyboard/timing items that automation can never fully verify.

### Visual Inspection (prioritized — color-contrast was found)

- **SC 1.4.3 / 1.4.11** — Verify the `.bio-link-sublabel` text
  (`#7f7773`/`#827a75` on `#f6eee3`, ~3.65–3.81:1) against the 4.5:1 minimum
  for normal text. This is the only contrast failure axe found, but check it
  in both light and dark theme (the page has a `ThemeToggle`) since the
  scan only covered the default theme.
- **SC 1.4.10 (Reflow)** — Re-check the bio tagline ("Engineer · Runner ·
  Soon-to-be / Father · SF Bay Area") at 320px width and at 200% browser
  zoom; this page was recently patched for a mobile word-break bug, so
  regression-check it specifically across a few device widths.
- **SC 1.4.12 (Text Spacing)** — Passed automated checks; spot-check once
  manually, no special focus needed.

### Screen Reader (prioritized — landmark/region issues were found)

- **SC 1.3.1 / 4.1.3** — The page lacks a `<main>` landmark and most
  content sits outside any named region (axe: `landmark-one-main`,
  `region`, 12 instances combined). With a screen reader (VoiceOver/NVDA),
  confirm whether the lack of landmarks makes it harder to skip to the
  link list vs. the header/portrait area.
- **SC 4.1.2 (Name, Role, Value)** — Passed automated checks for the
  profile image and link list; manually confirm each `BioLink` announces
  its label and sublabel together (e.g. "LinkedIn" vs. "mannyflo.com,
  Personal website") rather than just the icon.

### Keyboard Navigation

- **SC 2.4.3 / 2.4.7 (Focus Order / Focus Visible)** — Tab through all 7
  links plus the theme toggle; confirm a visible focus ring on each and a
  logical top-to-bottom order matching visual layout.
- **SC 2.1.2 (No Keyboard Trap)** — Low risk given no modals/menus on this
  page, but confirm Tab can reach and leave every interactive element.

### Cognitive

- **SC 2.4.6 (Headings and Labels)** — Only one `<h1>` ("Manny Flores") and
  no subheadings; this is appropriate for a link-in-bio page, no action
  needed.
- **SC 3.2.3 / 3.2.4 (Consistent Navigation/Identification)** — Single-page
  site, low risk; skip unless the site grows additional routes.

### Timing & Motion (always manual, cannot be automated)

- **SC 2.2.1 / 2.2.2 / 2.3.1** — The page uses Framer Motion fade/stagger
  animations on load. Confirm `prefers-reduced-motion` is honored (the
  code already calls `useReducedMotion()` and gates animation duration/
  offset accordingly — verify this manually with the OS-level reduced
  motion setting enabled, since automation cannot exercise that media
  query).
